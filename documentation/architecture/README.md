# Architecture — Cross-Cutting Documentation

This folder contains FlitX's cross-cutting architectural documentation — docs that describe the overall system design rather than any specific section like Fleet, Analytics, or Daily Program.

These documents are written for readers without prior programming or database experience. Technical terms are explained in plain English with analogies.

---

## Contents

| File | Description |
|---|---|
| [database-overview.md](./database-overview.md) | Introduction to how FlitX's database works, using plain-English analogies. Covers primary keys, foreign keys, and the single source of truth principle. |
| [data-relationships.md](./data-relationships.md) | Visual diagrams showing how every table in the FlitX database connects to every other table. |
| [crm-roadmap.md](./crm-roadmap.md) | The future customers table and CRM architecture, plus how `booking_contacts` will be migrated into it. |
